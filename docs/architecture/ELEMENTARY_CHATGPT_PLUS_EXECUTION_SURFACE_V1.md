**Status:** `CANONICAL_SUBORDINATE_CAPABILITY_ALIGNMENT`

## 1. Decision

ChatGPT Plus is adopted as an optional personal execution and interaction surface across ELEMENTARY. This is a cross-cutting capability update to the existing master plan, not a replacement master plan, second ERP, second control plane, second scheduler, second evidence authority or second source of truth.

The authority boundary remains unchanged:

- ELEMENTARY Construction OS owns canonical business truth;
- ADCP owns orchestration and governed task lifecycle;
- ELEMENTARY Agent Governance owns policy and admission;
- GitHub owns source, change, release and durable technical evidence;
- ChatGPT Plus provides account-dependent reasoning, research, coding, document creation, monitoring and connected-app execution surfaces.

A feature is usable only after four distinct states are evaluated: `PRODUCT_AVAILABLE`, `ACCOUNT_VISIBLE`, `PROJECT_VERIFIED` and `RUNTIME_ACTIVE`. Official product documentation is not runtime evidence.

## 2. What changes globally

Every ELEMENTARY chat, specialist task and governed workflow must use the strongest currently visible Plus capability when it removes manual work without weakening authority, privacy, evidence, rollback or portability.

## 3. Plan boundaries that must not be misrepresented

ChatGPT Plus does not include OpenAI API usage; API billing and controls are separate. Plus also does not provide the Business/Enterprise workspace governance layer: workspace agents, company knowledge, dedicated workspace, admin console, centralized app control or SSO.

The pricing page may show developer-mode availability, but full custom MCP applications with write/modify actions are currently documented for Business, Enterprise and Edu. Therefore:

1. do not infer full custom MCP write support from a generic developer-mode label;
2. do not recreate the existing Auth0/MCP implementation merely because a ChatGPT UI surface is missing;
3. preserve the canonical GitHub OIDC and governed cPanel MCP route;
4. use Plus plugins/apps where actually available and authorized;
5. classify direct custom MCP connection on Plus as unverified until the exact account UI and official plan boundary prove it.

Custom applications are not used by agent mode. Deep Research may use custom applications only for read/fetch operations. Sites and computer-use availability can vary by platform, rollout and region, including EEA limitations.

## 5. Operating rules for all future work

1. Load the live startup bundle and the ChatGPT Plus capability profile before plan or feature claims.
2. Recheck official OpenAI sources whenever plan, rollout, region, UI or capability may have changed.
3. Prefer direct connected tools over browser automation when the direct path is verified and sufficiently governed.
4. Prefer Work for multi-stage deliverables and Codex for repository coding; keep ADCP as orchestration authority.
5. Prefer Scheduled Tasks for recurring informational delivery and condition monitoring; do not duplicate a task already owned by ADCP or an external scheduler.
6. Use Deep Research for cited acquisition and synthesis, with specific-site restrictions where accuracy matters.
7. Treat memory, Projects and custom GPT knowledge as convenience context, not source of truth.
8. Require explicit confirmation before email sends, external messages, submissions, approvals, purchases, payments or destructive changes.
9. Register every durable document, spreadsheet, presentation, report, code change, evidence bundle or operational result.
10. Preserve a provider-neutral fallback so rollout loss, rate limits or plan changes do not block canonical operations.

## 6. Upgrade decision

The master business architecture does not require replacement. The required change is a master-wide capability and delivery alignment:

- add ChatGPT Plus as a governed execution surface;
- load its capability profile in every startup;
- update delivery choices in all stages;
- preserve the existing one-kernel, one-master, one-control-plane and zero-budget rules;
- keep Business/Enterprise as a future governance/economics decision, not an immediate dependency.

A later move to Business or Enterprise is justified only if team-wide custom MCP publishing, centralized app governance, company knowledge, workspace agents, SSO, compliance administration or stronger organizational data controls produce measurable value that exceeds the incremental cost.
